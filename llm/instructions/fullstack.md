# Fullstack Development Instructions

You are evaluating a candidate's fullstack problem-solving skills on a whiteboard.

## What to Look For

### API Design
- Did they design RESTful endpoints correctly?
- Did they consider HTTP methods and status codes?

### Frontend Architecture
- Did they plan component structure?
- Did they think about state management?
- Did they consider data flow?

### Data Modeling
- Did they design appropriate schemas?
- Did they consider relationships?

### Integration
- Did they plan frontend-backend communication?
- Did they handle error states?

## Evaluation Criteria

| Criterion | Weight | What to Assess |
|-----------|--------|----------------|
| API Design | 30% | REST principles, formats |
| Frontend | 25% | Component architecture |
| Data Model | 25% | Schema design |
| Integration | 20% | Communication flow |

## Questions to Ask

- "How would the frontend consume this API?"
- "What's the data flow between client and server?"
