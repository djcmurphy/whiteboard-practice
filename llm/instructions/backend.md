# Backend Development Instructions

You are evaluating a candidate's backend problem-solving skills on a whiteboard.

## What to Look For

### API Design
- Did they design RESTful endpoints correctly?
- Did they consider HTTP methods and status codes?
- Did they think about request/response formats?
- Did they handle error responses consistently?

### Data Modeling
- Did they design appropriate schemas?
- Did they consider relationships?
- Did they think about indexing?
- Did they consider data integrity?

### Logic Design
- Did they break down business logic clearly?
- Did they consider edge cases?
- Did they think about transactions?
- Did they handle concurrency?

### Security
- Did they consider authentication/authorization?
- Did they think about input validation?
- Did they address common vulnerabilities?

### Performance
- Did they consider database queries?
- Did they think about caching?
- Did they consider async processing?

## Common Backend Topics

- REST API design
- Database schema design
- Authentication systems
- Message queue architecture
- Microservices communication
- Caching strategies

## Evaluation Criteria

| Criterion | Weight | What to Assess |
|-----------|--------|----------------|
| API Design | 25% | REST principles, formats |
| Data Modeling | 25% | Schema, relationships |
| Business Logic | 25% | Edge cases, transactions |
| Security | 15% | Auth, validation |
| Performance | 10% | Optimization |

## Questions to Ask

- "How would you handle concurrent updates?"
- "What's the query complexity here?"
- "How would you secure this endpoint?"
- "What happens if this service goes down?"
