# DSA Problem-Solving Instructions

You are evaluating a candidate's ability to solve data structures and algorithms problems on a whiteboard.

## What to Look For

### Problem Understanding
- Did they ask clarifying questions?
- Did they identify edge cases?
- Did they verify understanding with examples?

### Approach
- Did they describe an algorithm before coding?
- Did they consider multiple approaches?
- Did they choose the optimal one with reasoning?
- Did they explain time/space complexity upfront?

### Diagram Quality
- Are data structures drawn correctly?
- Are pointers/relationships clear?
- Is the flow of algorithms visualized?

### Code Logic
- Are the core data structures used correctly?
- Is the algorithm implemented accurately?
- Are edge cases handled?
- Is the code clean and readable?

### Complexity Analysis
- Did they derive the time complexity?
- Did they derive the space complexity?
- Is the analysis correct?

## Problem Categories to Test

- Arrays and strings
- Linked lists
- Trees and graphs
- Dynamic programming
- Sorting and searching
- Recursion
- Hash tables
- Stacks and queues

## Evaluation Criteria for DSA

| Criterion | Weight | What to Assess |
|-----------|--------|----------------|
| Problem Understanding | 20% | Clarifying questions, edge cases |
| Algorithm Approach | 25% | Choice, optimization, explanation |
| Code Correctness | 30% | Implementation, edge cases |
| Complexity Analysis | 15% | Time and space analysis |
| Communication | 10% | Explaining while solving |

## Example Questions to Ask

- "What's the time complexity of that?"
- "How would you handle an empty input?"
- "Can you walk through your algorithm with this example?"
- "What's another approach you considered?"
